1. Visão Geral e Objetivo

Este documento estabelece as diretrizes técnicas para a implementação de um sistema autônomo de identificação de peças e faturamento industrial. A solução visa eliminar a dependência de memória visual humana, substituindo-a por um motor de visão computacional multimodal e uma lógica de negócio determinística.

Problema Central: Atualmente, a identificação de peças (ex: Cód. MEC 06332) é feita comparando fotos de celular com desenhos técnicos em PDF, gerando erros de faturamento e quebra do controle FIFO.

2. Arquitetura Multicamadas (Deep Dive)

Camada 1: Ingestão e Dicionário Geométrico (Ground Truth)

A inteligência não começa na foto, mas no PDF. Esta camada transforma o acervo estático de engenharia em um banco de dados de características pesquisáveis.

  • Conversão: Uso de bibliotecas Python (PyMuPDF) para converter vetores CAD em imagens rasterizadas de alta fidelidade.
  • Extração de Features: Utilização do Claude 3.5 Sonnet para "ler" a geometria. O foco não é o texto, mas a forma (furos, recortes em X, espessura, abas).
  • Indexação: Armazenamento das descrições em um banco vetorial (Vector Store) para recuperação rápida via RAG.

Camada 2: Agente Multimodal e RAG (O Motor de Decisão)

O ponto de entrada é o Telegram. O agente recebe a imagem do chão de fábrica e realiza a triangulação de dados.

Ação Processo Técnico
Localização Espacial Identifica onde estão as peças e onde está o texto escrito na foto.
Recuperação (RAG) Busca no banco vetorial as 5 descrições de PDFs mais próximas da geometria vista.
Match Geométrico O Claude compara a foto real com as 5 descrições e seleciona o código com maior probabilidade.

Camada 3: Validação Human-in-the-Loop (Segurança)

Nenhuma escrita em banco de dados ocorre sem aprovação. O bot envia uma mensagem estruturada com botões de confirmação/correção.

Camada 4: Motor de Regras FIFO (Execução Determinística)

Script Python puro que processa o lançamento. Ele ignora a IA nesta etapa para garantir precisão matemática 100%.

3. Roadmap de Construção (Passo a Passo)

1
Fase de Saneamento (Semana 1): Ingestão dos PDFs. Criar o dicionário visual. Sem isso, a IA não tem base de comparação confiável.
2
Desenvolvimento do Agente (Semana 2): Configuração do bot no Telegram e integração com OpenRouter. Criação do "Prompt de Geometria".
3
Implementação do Motor FIFO (Semana 3): Escrita da lógica em Python. Integração com a API do Google Sheets para manipular os saldos das NFs.
4
Homologação e Ajuste de Campo (Semana 4): Testes com iluminação real do galpão, sombras e pilhas de peças para calibrar a visão.
